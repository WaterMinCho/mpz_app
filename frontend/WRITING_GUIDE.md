# MPZ(마펫쯔) UX 라이팅 가이드

## 1. 브랜드 보이스

마펫쯔는 유기동물과 사람 사이의 따뜻한 연결을 만드는 서비스예요.
말투는 친구처럼 편안하지만 신뢰감을 잃지 않는 해요체를 써요.
어려운 상황에서도 사용자를 탓하지 않고, 다음 행동을 안내하는 데 집중해요.
동물을 "아이"라고 부르며, 입양이라는 무게감을 존중하되 무겁지 않게 전달해요.

---

## 2. 톤앤매너 원칙

### Do

- 해요체 통일 ("~해주세요", "~예요", "~있어요")
- 다음 행동을 함께 안내 ("다시 시도해주세요", "홈으로 돌아가기")
- 동물을 "아이"로 지칭 ("이 아이에 대한 게시물이 없어요")
- 짧고 명확한 문장 (한 문장에 하나의 정보)
- 긍정 프레이밍 우선 ("저장되었어요" > "저장 오류 없음")

### Don't

- 반말/합쇼체 혼용 ("됩니다", "했음" 등 섞어 쓰지 않기)
- 기술 용어 노출 ("서버 오류", "API 실패", "네트워크 에러", "401", "500" 등)
- 사용자 탓 표현 ("잘못된 입력", "올바르지 않은 요청")
- 느낌표 남용 (성공 메시지에 1회 정도만 허용)
- 과한 이모지 사용 (텍스트로 충분한 곳에 이모지 불필요)

## 3. 문구 레벨링

상황의 심각도에 따라 5단계로 나누고, 각 레벨에 맞는 톤을 적용해요.

### Level 1 — 일반 안내 (정보 전달)

> 차분하고 담백하게. 있는 그대로 전달.

| 상황 | 예시 |
|------|------|
| 빈 상태 | 아직 받은 알림이 없어요. |
| 빈 목록 | 아직 업로드된 게시글이 없어요. |
| 빈 검색 | 검색 결과가 없어요. 다른 키워드로 찾아보세요. |
| 빈 찜 | 찜한 동물이 없어요. 마음에 드는 아이를 찾아보세요. |
| 로딩 | 불러오는 중... |
| 안내 | 입양 절차와 관련된 동의서 내용을 확인해주세요. |

**톤 키워드:** 아직, ~없어요, ~해보세요

### Level 2 — 성공/완료 (긍정 피드백)

> 따뜻하게 축하하거나 확인. 느낌표는 핵심 성공 1회만.

| 상황 | 예시 |
|------|------|
| 저장 완료 | 저장되었어요. |
| 등록 완료 | 동물 정보가 등록되었어요. |
| 프로필 수정 | 프로필이 수정되었어요. |
| 입양 신청 | 입양 신청이 완료되었어요! |
| 링크 복사 | 링크가 복사되었어요. |
| 삭제 완료 | 삭제되었어요. |
| 로그아웃 | 로그아웃되었어요. |

**톤 키워드:** ~되었어요, ~완료되었어요

### Level 3 — 경고/주의 (주의 환기)

> 부드럽지만 명확하게. 무엇을 해야 하는지 같이 알려주기.

| 상황 | 예시 |
|------|------|
| 필수 항목 누락 | 필수 항목을 모두 입력해주세요. |
| 형식 오류 | 올바른 휴대폰 번호를 입력해주세요. |
| 인증 만료 | 인증번호가 만료되었어요. 다시 전송해주세요. |
| 삭제 확인 | 삭제된 정보는 되돌릴 수 없어요. |
| 로그인 필요 | 이 기능을 사용하려면 로그인이 필요해요. |
| 날짜 유효성 | 시작일이 종료일보다 늦을 수 없어요. |
| 나가기 확인 | 작성한 내용은 저장 및 복구가 불가능해요. |

**톤 키워드:** ~해주세요, ~수 없어요, ~필요해요

### Level 4 — 에러/실패 (문제 발생)

> 미안한 마음 + 재시도 안내. 원인을 기술 용어 없이 설명.

| 상황 | 예시 |
|------|------|
| 저장 실패 | 저장하지 못했어요. 다시 시도해주세요. |
| 불러오기 실패 | 정보를 불러오지 못했어요. 다시 시도해주세요. |
| 이미지 업로드 실패 | 이미지를 올리지 못했어요. 다시 시도해주세요. |
| 로그인 실패 | 로그인하지 못했어요. 다시 시도해주세요. |
| 찜하기 실패 | 찜하기에 실패했어요. 잠시 후 다시 시도해주세요. |
| 공유 실패 | 공유하지 못했어요. 다시 시도해주세요. |

**톤 키워드:** ~하지 못했어요, 다시 시도해주세요

### Level 5 — 치명적 에러 (서비스 불가)

> 상황을 솔직히 인정 + 대안 제시. 사용자 안심시키기.

| 상황 | 예시 |
|------|------|
| 페이지 없음 (404) | 앗, 여기는 아무도 없어요. 찾으시는 페이지가 사라졌거나 주소가 잘못되었을 수 있어요. |
| 서비스 장애 | 지금은 연결이 어려워요. 잠시 후 다시 방문해주세요. |
| 알 수 없는 오류 | 알 수 없는 문제가 생겼어요. 잠시 후 다시 시도해주세요. |

**톤 키워드:** 앗, 지금은, 잠시 후

## 4. 상황별 문구 패턴

### 4.1 빈 상태 (Empty State)

```
패턴: 아직 {대상}이/가 없어요. (+ 선택적 행동 유도)

예시:
- 아직 받은 알림이 없어요.
- 아직 업로드된 게시글이 없어요.
- 찜한 센터가 없어요. 관심 가는 센터를 찾아보세요.
- 등록된 동물이 없어요.
```

### 4.2 확인 다이얼로그 (Confirm)

```
패턴:
  제목: 정말 {행동}하시겠어요?
  설명: {결과 안내}

예시:
- 제목: 정말 삭제하시겠어요?
  설명: 삭제된 정보는 되돌릴 수 없어요.

- 제목: 정말 로그아웃하시겠어요?
  (설명 없이 버튼만으로 충분)

- 제목: 정말 철회하시겠어요?
  설명: 진행 중인 입양 절차가 취소돼요.
```

### 4.3 에러 토스트

```
패턴: {동작}하지 못했어요. 다시 시도해주세요.

예시:
- 저장하지 못했어요. 다시 시도해주세요.
- 정보를 불러오지 못했어요. 다시 시도해주세요.
- 이미지를 올리지 못했어요. 다시 시도해주세요.
```

### 4.4 성공 토스트

```
패턴: {대상}이/가 {동작}되었어요.

예시:
- 프로필이 수정되었어요.
- 동물 정보가 등록되었어요.
- 링크가 복사되었어요.
- 입양 신청이 완료되었어요!  (핵심 액션에만 느낌표)
```

### 4.5 로딩

```
패턴: {동작}하는 중...

예시:
- 불러오는 중...
- 로그인 중...
- 이미지를 올리는 중...
- 저장하는 중...
```

### 4.6 입력 placeholder

```
패턴: {대상}을/를 {동작}해주세요. 또는 {대상}(으)로 검색해보세요.

예시:
- 이름을 입력해주세요.
- 품종명으로 검색해보세요.
- 자유롭게 작성해주세요.
- 댓글을 남겨보세요.
```

### 4.7 로그인 유도

```
패턴:
  제목: 로그인이 필요해요.
  설명: 이 기능을 사용하려면 로그인해주세요.
  버튼: 로그인하기 / 닫기
```

## 5. 금지 표현

| 금지 | 대체 |
|------|------|
| 서버 오류가 발생했습니다 | 지금은 연결이 어려워요 |
| API 호출에 실패했습니다 | 정보를 불러오지 못했어요 |
| 네트워크 에러 | 연결 상태를 확인해주세요 |
| 잘못된 입력입니다 | 입력 내용을 다시 확인해주세요 |
| 올바르지 않은 요청 | 다시 시도해주세요 |
| 권한이 없습니다 | 이 기능을 사용할 수 없어요 |
| 인증되지 않은 사용자 | 로그인이 필요해요 |
| null / undefined / error | (절대 노출 금지) |
| 실패했습니다 | ~하지 못했어요 |
| 오류가 발생했습니다 | 문제가 생겼어요 |
| ~했습니다 (합쇼체) | ~했어요 (해요체) |
| ~없습니다 (합쇼체) | ~없어요 (해요체) |

## 6. 현재 코드 톤 분석 및 개선 포인트

현재 코드베이스 분석 결과, 아래와 같은 일관성 이슈가 발견됨.

### 6.1 합쇼체/해요체 혼용

현재 상당수 메시지가 합쇼체("-습니다")와 해요체("-어요")를 혼용하고 있음.

```
현재: "로그인에 실패했습니다. 다시 시도해주세요."  (합쇼체)
개선: "로그인하지 못했어요. 다시 시도해주세요."     (해요체)

현재: "사용자 정보를 가져올 수 없습니다."           (합쇼체)
개선: "사용자 정보를 불러오지 못했어요."             (해요체)

현재: "입양 신청이 완료되었습니다!"                  (합쇼체)
개선: "입양 신청이 완료되었어요!"                    (해요체)
```

### 6.2 기술 용어 노출

```
현재: "잘못된 요청 형식입니다."
개선: "다시 시도해주세요."

현재: "상태 변경 중 오류가 발생했습니다."
개선: "상태를 변경하지 못했어요. 다시 시도해주세요."

현재: "질문을 불러오는 중 오류가 발생했습니다. 콘솔을 확인해주세요."
개선: "질문을 불러오지 못했어요. 다시 시도해주세요."
```

### 6.3 부정 프레이밍

```
현재: "센터 찜을 실패했습니다."
개선: "찜하지 못했어요. 다시 시도해주세요."

현재: "새로고침에 실패했습니다."
개선: "새로고침하지 못했어요. 다시 시도해주세요."
```

### 6.4 잘 된 예시 (유지)

```
- "앗, 여기는 아무도 없어요" (404 페이지)
- "아직 업로드된 게시글이 없어요." (빈 상태)
- "삭제된 정보는 되돌릴 수 없어요." (삭제 확인)
- "작성한 내용은 저장 및 복구가 불가능해요." (이탈 확인)
- "아직 이 아이에 대한 게시물이 없어요" (동물 관련 빈 상태)
```

이 예시들은 해요체, 따뜻한 톤, 명확한 정보 전달이 잘 지켜진 문구임.
